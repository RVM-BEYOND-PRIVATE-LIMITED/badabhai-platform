# BadaBhai Worker App (Flutter)

Phase 1 **scaffold** of the chat-first worker app. Intentionally lightweight: no
heavy UI, no extra pub dependencies, and the API client returns mock data so the
flow runs end-to-end before the backend is wired.

## Flow / screens

`Splash → PhoneLogin → OtpVerify → Consent → ChatProfiling → (VoiceNote) →
ProfilePreview → ResumePreview`

```
lib/
  main.dart              # entrypoint
  app.dart               # MaterialApp + routes
  router.dart            # Routes constants + route table
  core/
    theme.dart           # Material 3 theme
    api/api_client.dart   # PLACEHOLDER API client (mock; no HTTP dep yet)
    state/app_state.dart  # ChangeNotifier holding ids (no raw PII beyond phone)
  features/
    splash/  auth/  consent/  chat/  voice/  profile/  resume/
test/
  widget_test.dart
```

## Run (requires the Flutter SDK)

```bash
cd apps/worker-app
flutter pub get
flutter analyze
flutter test
flutter run --dart-define=API_BASE_URL=http://10.0.2.2:3001   # Android emulator -> host
```

> ⚠️ Flutter is **not installed** on the current machine, so `flutter pub get` /
> `analyze` / `test` could not be run here. The Dart sources are written to pass
> `flutter analyze` with `flutter_lints`. Install the Flutter SDK to build/run.
>
> The platform runner folders (`android/`, `ios/`, `web/`, etc.) are generated by
> `flutter create .` in this directory — run it once after installing the SDK.
